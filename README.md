    设计原则：
        1、变化隔离，找出变化，分开变化和不变的
            隔离，封装变化的部分
        2、面向接口，依赖倒置
            使用者使用接口，提供者实现接口，接口可以是超类
        3、开闭原则，对修改关闭，对扩展开放
        4、多用组合，少用继承
        5、最少知道原则，迪米特法则
        6、单一职责原则。
    第一、找出变化
    第二、接口
    第三、组合，一个关系，多个关系，链式关系。
